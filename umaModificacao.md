# Como usar o Git

## O que é o Git?
O Git é uma ferramenta de versionamento de código,
em outras palavras, é uma ferramenta criada para nos ajudar
a criar versões diferente para o nosso código
pondendo atualiazar o nosso código, voltar a versão anterior,
criar uma ***branch*** e trabalhar nela isoladamente, dentre 
outras funcionalidades.\
Imagine o nosso projeto como uma linha do tempo reta, onde cada 
modificação representa um ponto novo nessa linha do tempo, bom isso
seria possível mas, na realidade isso não é prático.\
Pense, e se o seu novo código tiver algum bug? Como você pode voltar
a versão anterior se já atualizou? Como você pode criar uma parte 
esperimental sem interferir no código principal?.

## Como usar o Git
### Termos utilizados no Git

**Main/Master** = É a ramificação/linha do tempo principal do nosso projeto,
nela conta o código que está ativo/funcinando.

**Branch** = É uma ramificação paralela do nosso código/projeto, nela
podemos trabalhar isoladamente e não irá modificar nada no nosso código 
principal.

**Merge** = É quando pegamos uma ***branch*** e a introduzimos na nossa ***main***
e apartir desse momento ela faz parte do nosso código principal.

**Readme.md** = Não é algo utilizado no Git, mas relacionado, práticamente
todo arquivo criado deverá ter um arquivo "Readme.md", é como se fosse um "leia-me"
nesse arquivo, está todas as instruções de como funciona o seu código ou para que ele
serve.

**Repositório** = Nada mais é do que um local onde armazenamos todos os arquivos
de um projeto em que estamos trabalhando.
